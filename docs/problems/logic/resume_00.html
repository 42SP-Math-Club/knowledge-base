Apresentações

A sessão começou com cada membro apresentando seu nível de contato com a matemática — do ensino médio a graduações em exatas. Não houve hierarquia nisso: o objetivo era entender de onde cada um parte, não classificar quem sabe mais.

Essa diversidade foi imediatamente útil. Ao longo da sessão, quem tinha mais base ajudava naturalmente quem tinha menos, sem que isso precisasse ser organizado.

O Problema

O problema da sessão foi uma representação matemática de Bit Masking — tema diretamente ligado ao cursus da 42.

Seja \(U\) um conjunto universal finito com cinco variáveis: \[U = \{x_0,\, x_1,\, x_2,\, x_3,\, x_4\}\]

Seja \(S\) qualquer subconjunto de \(U\) (ou seja, \(S \subseteq U\)). Definimos uma função característica \(f(S)\) que mapeia um conjunto \(S\) para uma string binária de 5 bits.

A função avalia os elementos de \(S\) usando uma condição lógica para gerar cada bit \(b_i\) da string \(b_4b_3b_2b_1b_0\): \[b_i = \begin{cases} 1 & \text{se } x_i \in S \\ 0 & \text{se } x_i \notin S \end{cases}\]

(Nota: \(b_0\) representa o bit menos significativo, à direita, correspondente a \(x_0\).)

Dados os subconjuntos: \[A = \{x_0,\, x_2,\, x_3\} \qquad B = \{x_1,\, x_2,\, x_4\}\]

  1. Qual é \(f(A)\)?

  2. Qual é \(f(B)\)?

Como a Sessão Aconteceu

Ninguém sabia a resposta de antemão. O que aconteceu foi uma pesquisa matemática coletiva e lenta — no bom sentido. Cada pessoa precisava externalizar seus pensamentos, suas ideias e, principalmente, suas dúvidas.

“Nah, eu não sabia qual símbolo usar” — então só escreve por extenso mesmo, tipo: “3 está no conjunto dos naturais”. A ideia é externalizar o que sabe, para assim os outros poderem aprender, ajudar e acompanhar.

Quando a notação faltava, o grupo simplesmente inventava a própria. A formalização veio depois, como abstração do que já havia sido pensado — da mesma forma que a matemática foi construída historicamente.

Não houve professor. Não houve cronômetro. O modelo foi o da Grécia antiga: pensar em voz alta, divergir, questionar e reconstruir o raciocínio coletivamente.

O Raciocínio

O raciocínio foi construído em três etapas, cada uma surgindo naturalmente da anterior.

Etapa 1 — Representação Binária (Herom)

Herom começou desenhando e indagando: como representar formalmente se um elemento pertence ou não a um conjunto? A resposta imediata foi a função característica binária, que o grupo escreveu assim:

\[b_i = \begin{cases} 1 & \text{se } x_i \in S \\ 0 & \text{se } x_i \notin S \end{cases}\]

Esse é exatamente o \(f(S)\) do problema. Cada bit \(b_i\) registra a presença ou ausência de \(x_i\) no subconjunto \(S\).

Etapa 2 — Generalização para Base 3 (Breno)

A primeira ideia de generalização veio do Breno: e se o conjunto \(S\) fosse ordenado? Um elemento poderia estar presente de duas formas distintas — presente sem ordem, ou presente e sendo o elemento de maior posição. Isso naturalmente pede uma terceira condição, e portanto uma base 3:

\[T_i = \begin{cases} 0 & \text{se } x_i \notin S \\ 1 & \text{se } x_i \in S \text{ e não está ordenado} \\ 2 & \text{se } x_i \in S \text{ e está ordenado (é o último)} \end{cases}\]

Com isso, a string resultante deixa de ser binária e passa a ser ternária — cada dígito carrega mais informação sobre a estrutura do conjunto.

Etapa 3 — Generalização para Base \(m\) (Felipe / atoi_base)

A partir da ideia do atoi_base trazida por Felipe — função que converte strings para inteiros em bases arbitrárias — o grupo percebeu que as etapas anteriores eram casos particulares de uma função mais geral.

Para qualquer base \(m\), define-se \(R_i\) como:

\[R_i: \begin{cases} n = \dfrac{x_i}{m} \\[6pt] n_i = 0 & \text{se } x_i \notin S \\[4pt] n_i = 1 & \text{se } x_i \in S \text{ (sendo } m \text{ a base de } S\text{)} \\[4pt] n_j = \dfrac{x_j}{m} & \text{se } x_i \text{ for o último elemento} \end{cases}\]

O exemplo produzido no quadro foi \(\{x_1, x_0, x_3\}\), aplicando \(R_i\) sobre esse subconjunto ordenado.

A conexão com atoi_base foi o que permitiu enxergar a estrutura geral: assim como atoi reconstrói um número a partir de seus dígitos em uma base qualquer, \(R_i\) constrói uma representação posicional a partir da pertinência e ordem dos elementos em \(S\).

O problema, a generalização para base 3 e a função \(R_i\) foram inteiramente inventados pelo grupo durante a sessão. Nenhuma referência externa foi consultada.

Nota: A Etapa 3 não foi verificada. O grupo chegou até essa formulação, mas ainda não houve tempo para pensar com cuidado se ela está correta ou completa. Pode ser que fique como ponto de partida para a próxima sessão.

Observações sobre o Formato

Esta primeira sessão também serviu para apresentar o Estatuto do clube e discutir sua filosofia. Alguns pontos que emergiram na prática:

  • Pessoas sem base matemática formal conseguem fazer perguntas que avançam a discussão tanto quanto quem tem;

  • A ausência de pesquisa durante a sessão não foi uma limitação — foi o que tornou o pensamento genuinamente coletivo;

  • Inventar notação antes de formalizá-la revelou por que a notação matemática existe do jeito que existe.

A ideia é aprender do “por que” a matemática foi desenhada do jeito que é hoje — descobrindo, igual como os matemáticos antigamente faziam.
